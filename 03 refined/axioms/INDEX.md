# Axioms Index

> **Axioms** are the user's core cognitive filters — deep philosophical principles distilled from personal experience.
>
> Unlike Skills (the execution layer, "how to do it"), Axioms operate at the thinking layer ("why"), used to inspire deep reflection and provide alternative perspectives.

---

## How to Use

### Quick Browse (INDEX)
**When to read the index**:
- When starting a task involving strategic decisions or architectural design
- When you want to understand the user's "cognitive baseline"
- When you need to find axiom categories relevant to the current task

### Deep Reading (detailed files)
**When to read in detail**:
- When a task touches the user's core values or methodology
- When you need to understand a particular axiom's "deep implications" and "application criteria"
- When the user explicitly asks to reference their personal philosophy
- When doing reflective work (distilling axioms, deep retrospectives)

### Core Axiom Clusters

These axioms form an organic whole and are often invoked together:

- **AI Management Triangle**: A02 (multiplier) + A03 (IC→Manager) + A04 (reliability) — the foundational mindset for all AI collaboration

### Proactive Retrieval Triggers

When the user mentions the following keywords, proactively consult the relevant axiom:

- "managing AI", "delegation" → A03, A04
- "verification", "trust", "reliability" → A04
- "context", "memory", "documentation" → A05
- "builder", "tool extension", "building" → A06
